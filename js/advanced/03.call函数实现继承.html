03.call函数实现继承

大约 4 分钟

1.call()函数在javaScript中可以实现对象继承

function Product(name, price) {
  this.name = name;
  this.price = price;
}

function Food(name, price) {
  Product.call(this, name, price);
  this.category = 'food';
}

const aa = new Food('cheese', 5)
console.log(aa.name);
console.log(aa.category);
  • 这段代码演示了如何通过函数构造继承（function constructor inheritance）在 JavaScript 中实现对象继承。让我们一步步来分析：

1.1 Product 构造函数

function Product(name, price) {
  this.name = name;
  this.price = price;
}
  • 这是一个构造函数，Product 用来创建带有 name 和 price 属性的对象。
  • 使用 this 关键字将传入的 name 和 price 参数赋值给新创建的对象(new Food())。

1.2 Food 构造函数

function Food(name, price) {
  Product.call(this, name, price);
  this.category = 'food';
}
  • 这是另一个构造函数，Food 用来创建带有 name、price 和 category 属性的对象。
  • 通过 Product.call(this, name, price)，Food 构造函数调用了 Product 构造函数，并把当前的 this 传递给它。 这意味着，Food 构造函数可以借用 Product 构造函数来给当前对象添加 name 和 price 属性。这种模式就是在 JavaScript 中通过 call 方法实现继承。
  • 最后，this.category = 'food'; 给当前对象添加了一个新属性 category，并将它设为 'food'。

1.3 创建实例

const aa = new Food('cheese', 5);
  • 这里使用 new 关键字调用 Food 构造函数，创建了一个名为 aa 的新对象。
  • 在这个过程中，this 被绑定到新创建的对象实例，Product.call(this, name, price) 通过 Product 构造函数为新对象设置了 name 和 price 属性，而 this.category = 'food'; 则为它添加了 category 属性。

1.4 输出结果

console.log(aa.name);     // 输出 'cheese'
console.log(aa.category); // 输出 'food'
  • aa.name 输出 'cheese'，因为它在 Product 构造函数中被赋值。
  • aa.category 输出 'food'，因为它在 Food 构造函数中被赋值。

1.5 总结

  • Food 继承了 Product 的属性 name 和 price，并且自己添加了一个新的属性 category。
  • Product.call(this, name, price) 通过显式调用 Product 构造函数，实现了对父类构造函数的继承。

这样就实现了一个通过构造函数进行继承的简单例子。

2. 问题：为什么通过Product.call(this, name, price) 通过显式调用 Product 构造函数，Product.call(this, name, price)中这个this是绑定到哪里?

在 JavaScript 中，函数本质上是对象，函数对象有一个内置的方法叫做 call。通过 call 方法，可以显式地调用一个函数，并为它绑定执行上下文，也就是 this。

2.1 为什么可以调用 Product 函数？

  • 在 JavaScript 中，函数也是一种对象，因此 Product 这个构造函数可以像普通函数一样被调用。构造函数只是一个使用 new 关键字创建实例的特殊函数，但是也可以像调用普通函数那样直接调用它。

  • 通过 Product.call(this, name, price)，可以直接调用 Product 函数，并且传递 this 指向当前的上下文，允许你在当前的上下文中执行 Product 函数的逻辑。

2.2 Product.call(this, name, price) 中的 this 是绑定到哪里的？

call 方法允许你显式地指定 this 的上下文。换句话说，可以控制函数执行时的 this 指向。
具体来说，Product.call(this, name, price) 做了以下几件事：

  • this 是当前 Food 构造函数正在创建的新对象（new Food() 时创建的对象）。
  • call 方法的第一个参数决定了函数调用时的 this 指向。因此，当写 Product.call(this, name, price) 时，你告诉 JavaScript 在调用 Product 构造函数时，this 应该指向当前的 Food 构造函数正在创建的新对象。
  • 由于 this 现在指向新创建的 Food 对象，所以 Product 构造函数内的 this.name = name 和 this.price = price 会为这个新对象（即 aa 实例）添加 name 和 price 属性。

2.3 this 的绑定过程

让我们分步骤看一下这个 this 是如何绑定的：

  • 当 new Food('cheese', 5) 被调用时，new 操作符会：

    1. 创建一个空对象 aa。
    2. 将 this 绑定到这个空对象上，即 this 在 Food 构造函数中指向 aa。

  • 调用 Product.call(this, name, price) 时：

    • this 指向的是当前的 aa 对象。
    • call 方法会将 this 传递给 Product 构造函数，保证 Product 构造函数中的 this 仍然指向 aa，从而将 name 和 price 属性赋值给 aa。

举例说明

function Product(name, price) {
  this.name = name;
  this.price = price;
}

function Food(name, price) {
  // 这里的 this 是 new Food() 生成的新对象
  console.log(this);  // 输出新创建的对象 (aa)
  
  // 调用 Product，并将 this 绑定为当前的 this (即 aa)
  Product.call(this, name, price);
  
  // 给当前对象 (this) 添加新的属性
  this.category = 'food';
}

const aa = new Food('cheese', 5);
console.log(aa.name);     // 输出 'cheese'
console.log(aa.category); // 输出 'food'

在 Product.call(this, name, price) 中的 this 被绑定到 aa，即 new Food() 创建的对象。因此，Product 的构造函数内部的 this.name = name 和 this.price = price 实际上是在为 aa 添加 name 和 price 属性。

总结：

  • Product.call(this, name, price) 调用 Product 函数，并把 this 绑定到当前 Food 构造函数中的 this，也就是新创建的 aa 对象。
  • this 在 call 中是指向 new Food() 创建的对象，确保 Product 的属性和方法可以应用到这个新对象上。

3. 应用

//hasOwnProperty 是 Object.prototype 上的一个普通方法。
function assignValues (target, source) {
  for (let key in target) {
    if (Object.prototype.hasOwnProperty.call(source, key)) {
      target[key] = source[key]
    }
  }
  return target
}
