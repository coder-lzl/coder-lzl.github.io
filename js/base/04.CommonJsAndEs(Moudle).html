04.CommonJs和Es(Moudle)导入与导出

大约 7 分钟
此页内容
  • 一、CommonJs的导入与导出
  • 二、ES(Module)的导入与导出
  • 三、CommonJS 模块 和 ES Modules（ESM） 的不同工作方式

在javaScript中，CommonJS和ES(Module)是两种常见的模块系统，它们在导入和导出中有着不同的方法和工作方式。

一、CommonJs的导入与导出

CommonJS导入导出主要由：module.exports、exports、require组成。

  1. module.exports:

    • module.exports是CommonJs模块系统中真正的导出对象.
    • 当你需要导出一个对象、函数、类等时，必须直接需修改module.exports.
    • 完全覆盖：如果你将module.exports赋值为一个新的对象或值，它会完全替换掉原本的exports对象.

  2. exports:

    • exports是module.exports的一个引用.
    • 默认情况下，exports是module.exports的别名，二者指向同一个对象。因此。可以通过exports来向外暴露模块接口.
    • 附加属性：exports是用来向module.exports追加属性和方法的。它的作用主要是向module.exports添加新的属性或方法，而不能直接将其赋值为一个新的对象，否则会导致exports断开与module.exports的关联.

  3. require():

    • require()是CommonJs模块系统中用于导入模块的函数.
    • 当使用require()导入一个模块时，它会加载并执行该模块的代码，并返回该模块导出的module.exports对象.
    • **缓存机制:**在第一次调用require()时,模块会被执行并缓存；之后再调用require()导入相同的模块时，直接返回缓存中的module.exports，避免重复加载和执行模块代码.
    • require()可以导入各种类型的模块，包含对象、函数、类等。它会返回模块通过module.exports或exports导出的内容.

  4. 示例：

    错误做法：

    //module.exports和exports的错误做法
// math.js
const multiply =  function(a, b) {
    return a * b;
  }
module.exports = {
  multiply
};

// 使用 exports 添加额外的方法
exports.add = function(a, b) {
  return a + b;
};

exports.subtract = function(a, b) {
  return a - b;
};


//require()
// main.js
const { add, subtract, multiply } = require('./math'); // 解构导入需要的方法
console.log(add(2, 3));      // 输出 add is not a function
console.log(subtract(5, 3));   // 输出 subtract is not a function
console.log(multiply(2, 3));   // 输出 6

    module.exports 一旦被重新赋值，exports 就会断开与它的引用关系，因此通过 exports 添加的属性不会生效。 最好使用 module.exports 一次性导出所有需要的内容，或者使用 exports 添加属性，但不要同时混合使用它们。

    正确做法：

    // math.js
const add = function(a, b) {
  return a + b;
};

const subtract = function(a, b) {
  return a - b;
};

const multiply = function(a, b) {
  return a * b;
};

module.exports = {
  add,
  subtract,
  multiply
};

//require()
// main.js
const { add, subtract, multiply } = require('./math'); // 解构导入需要的方法
console.log(add(2, 3));        // 输出 5
console.log(subtract(5, 3));   // 输出 2
console.log(multiply(2, 3));   // 输出 6

//或
const mathModule = require('./math')
console.log(mathModule.add(2, 3));        // 输出 5
console.log(mathModule.subtract(5, 3));   // 输出 2
console.log(mathModule.multiply(2, 3));   // 输出

二、ES(Module)的导入与导出

ES(Module)导入导出主要由：export default、export、import组成.

  1. export default:

    • **默认导出：**使用export default导出一个模块的默认内容，通常是一个对象、函数或类.

    • **每个模块只能有一个默认导出：**一个模块只能有一个export default，表示模块的主要内容；在导入时，你可以给它起一个自定义的名称，名字可以与导出的内容名称不同.

    • 示例：

      // 导出的模块 (math.js)
export default function add(a, b) {
  return a + b;
}

// 导入的文件 (index.js)
//上面导出的是add函数，我们再导入的时候名字可以不同
import addFunction from './math';
console.log(addFunction(1, 2)); // 3

另一种导入方式（不使用自定义名称）
import * as math from './math';
console.log(math.add(1, 2)); // 3
如果使用 export default 导出一个模块内容，import * as math 会导入整个模块作为一个对象，并通过 math 访问所有的导出内容。

  2. export:

    • 命名导出：可以使用export导出多个具名的成员（变量、函数、类等）供外部使用.

    • 多次导出：一个模块中可以由多个export，每个export都导出不同的成员；在导入时，必须使用相同的名称来访问这些导出的内容。

    • **导入：在导入具名导出的模块时，你必须使用 **大括号{} 并且保持导出时的名称一致。

    • 示例：

      // 导出的模块 (math.js)
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;

// 导入的文件 (index.js)
import { add, subtract } from './math';

console.log(add(1, 2));      // 3
console.log(subtract(2, 1)); // 1

  3. 混合导入（默认导出 + 命名导出）:

    // 导出的模块 (math.js)
export default function multiply(a, b) {
  return a * b;
}

export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;

// 导入的文件 (index.js)
import multiplyFunction, { add, subtract } from './math';  // 混合导入

console.log(multiplyFunction(2, 3)); // 6
console.log(add(1, 2));              // 3
console.log(subtract(5, 3));         // 2

  4. 导入整个模块（import * as）：

    如果想要导入模块中的所有导出，可以使用 import * as 语法。这会将模块的所有导出整合成一个对象，该对象访问各个成员。

    // 导出的模块 (math.js)
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;

// 导入的文件 (index.js)
import * as math from './math';  // 导入整个模块作为一个对象

console.log(math.add(1, 2));      // 3
console.log(math.subtract(5, 3)); // 2

  5. export default和export的不同行为：

    • export default：

    • 错误解构：解构导入时直接提取属性（X）

      可以直接在导入时使用解构来提取对象的属性。这样做是 不允许的，因为 export default 导出的对象本身就是整个对象，并且只能以一个整体导入。

      // 错误的做法
import { name, age } from './module'; // 报错：无法解构默认导出的对象

      为什么会报错： export default 导出的内容是整个对象，而不是一个具名的导出，因此不能直接在导入时使用 {} 解构对象的属性。只能先导入整个对象，再进行解构。

    • 正确解构：

      假设我们有一个对象作为默认导出：
// module.js
const user = {
  name: 'Alice',
  age: 30
};
export default user;

解构导入
当你导入这个默认导出的对象时，可以直接使用 解构 来提取对象的属性。
// index.js
import user from './module';
const { name, age } = user;
console.log(name); // Alice
console.log(age);  // 30

    • export：具名导出，可以在导入时直接解构。

三、CommonJS 模块 和 ES Modules（ESM） 的不同工作方式

  1. module.exports和export default的本质区别：

    CommonJS(module.exports):

    • 在CommonJS中，module.export是一个指向对象或函数的引用，可以在模块中设置module.exports为任何类型的值，包括对象、数组、函数等。当使用require()导入时，直接获取到的是module.exports设置的对象.

    ES Moduels(module.exports):

    • 在ES modules中，export default导出默认内容本质上是整个模块默认导出对象，这和module.exports有些相似，但是它的导入方式和CommonJs的工作方式略有不同。在ESM中**，export default是单一的默认内容**，可以是对象、函数、类等，而导入时，需要使用import语法并给这个默认导出的内容起一个名称。

  2. module.exports 导出的对象与解构

    // math.js (CommonJS 模块)
module.exports = {
  name: 'Alice',
  age: 30
};

导入和解构（require()）
// index.js (CommonJS 导入)
const { name, age } = require('./math');
console.log(name); // Alice
console.log(age);  // 30
    • 原因：在 CommonJS 中，require() 直接返回 module.exports 对象。然后，require 的返回值本身就是一个 对象，可以直接对这个对象进行解构。

  3. export default 导出的对象与解构

    // math.js (ES6 模块)
const user = {
  name: 'Alice',
  age: 30
};

export default user;

导入和解构（import）
// index.js (ES6 导入)
import user from './math';
const { name, age } = user;
console.log(name); // Alice
console.log(age);  // 30

    原因：在 ES6 模块中，import user from './math' 先 导入整个默认导出的对象，然后你可以对它进行解构。不能直接在导入时进行解构（例如：import { name, age } from './math';），因为 export default 导出的内容是模块的 整个对象.

  4. 为什么 CommonJS 可以直接解构，ES6 不能

    • CommonJS 使用 module.exports 导出的对象是 整个模块的内容，当你通过 require() 导入时，你获得的是这个导出的 对象本身。因此，解构是直接从这个对象上进行的。
    • ES6 模块 使用 export default 导出的默认内容也是一个对象，但在导入时，import user from './module' 导入的 是整个默认导出的对象，你并不能在导入时直接解构它。你必须先导入整个对象，再在后续使用解构来提取属性。
